Works with your tools

The same SKILL.md file works across every major AI coding tool. Only the loading mechanism changes.

Claude Code Most features

Native auto-routing — skills load automatically from natural language. No @mentions needed.

Install
terminal
# Global — available in every project ./scripts/install-skills.sh --global # Project-local — only this repo ./scripts/install-skills.sh # Extended skills ./scripts/install-skills.sh --global --dir extended-skills
Where skills live
paths
# Global ~/.claude/skills/<name>/SKILL.md # Project-local .claude/skills/<name>/SKILL.md
How to invoke
Claude Code prompt
# Natural language — auto-routes "review my code" → /code-review "plan this feature" → /planning "write a ticket" → /ticket-writer "debug this error" → /debugging # Direct slash command /code-review /sprint-planning
Claude Code gets the most. Auto-routing, agent PM (/daily-checkin + /team-digest), and the install script all require Claude Code. Every other tool gets skill content but loads it manually.
Agent PM setup — no PM needed
1
Copy templates/TEAM.md and templates/SPRINT.md to your project root
2
Fill in SPRINT.md with tasks and owners
3
Each dev runs /daily-checkin once a day — 30 seconds, reads real git log
4
Run /team-digest any time to get sprint summary, velocity, and blocker table
Installed? Go deeper.
Sub-agents, MCP, workflow scripts, memory architecture, hooks, and eval harnesses — Claude-specific patterns beyond install.

Cursor Full support

Skills export as .mdc rules in .cursor/rules/. @mention in chat to activate.

Export & install
terminal
# Export all skills for Cursor ./scripts/export-skills.sh cursor # Creates .cursor/rules/<name>.mdc # for each skill
How to invoke
Cursor chat
# @mention the rule name @code-review # Or describe naturally "review this for security issues" # Cursor reads the .mdc file # and applies the skill automatically
Skills appear in Cursor's rule picker after export. All 24 dev-pm skills are supported. The rule file contains the full SKILL.md content — no truncation.
💙

VS Code + Copilot 3 options

Works with GitHub Copilot, Cline, and Continue. Each loads skills differently.

GitHub Copilot
Always-on via .github/copilot-instructions.md or on-demand with #file:
Cline
Always-on via .clinerules in project root
Continue
Always-on via .continuerules in project root
GitHub Copilot — always-on
terminal
./scripts/export-skills.sh copilot # Appends top skills to # .github/copilot-instructions.md # Keep this file under 400 lines # or quality degrades
GitHub Copilot — on-demand
Copilot Chat
# Reference the skill file directly #file:.github/skills/code-review.md Review this function for issues
Cline or Continue
terminal
# For Cline ./scripts/export-skills.sh cline # Creates .clinerules # For Continue ./scripts/export-skills.sh continue # Creates .continuerules
See docs/vscode-setup.md in the repo for a detailed comparison of all three options and when to use each.

OpenAI Codex Full support

Skills export as playbooks in .codex/playbooks/. Reference them in task descriptions.

Export & install
terminal
# Export all skills for Codex ./scripts/export-skills.sh codex # Creates .codex/playbooks/<name>.md
How to invoke
Codex task
# Reference by name in task "Follow the code-review playbook to review this PR" # Or by path "Use .codex/playbooks/security.md to audit this endpoint"
Codex reads the full playbook file when referenced. The skill content — severity tables, output templates, worked examples — all transfer intact.
🏄

Windsurf Cascade rules

Skills export as Cascade rules in .windsurf/rules/. Prompt Cascade to follow a rule.

Export & install
terminal
# Export all skills for Windsurf ./scripts/export-skills.sh windsurf # Creates .windsurf/rules/<name>.md
How to invoke
Windsurf / Cascade
# Tell Cascade to follow the rule "Follow the code-review procedure for this file" # Or reference directly "Use .windsurf/rules/security.md to audit this endpoint"
Windsurf Cascade reads the rule file when prompted. The full skill content — output templates, severity tables, worked examples — applies to the current session.

Gemini CLI Slash commands

Skills export to .gemini/skills/. Invoke with /skill-name or describe naturally.

Export & install
terminal
# Export all skills for Gemini CLI ./scripts/export-skills.sh gemini # Creates .gemini/skills/<name>.md
How to invoke
Gemini CLI
# Slash command /code-review # Or natural language "review this for security issues" # Skills auto-load from # .gemini/skills/ directory
Gemini CLI reads skill files from the .gemini/skills/ directory. All 24 dev-pm skills export cleanly.
